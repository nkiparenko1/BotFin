# PRD: AI Финансовый Советник (MVP)

**Версия:** MVP 1.0  
**Дата:** Июнь 2026  
**Статус:** Реализовано в коде (~85%), см. [STATUS.md](./STATUS.md)  
**Платформа:** Web (Next.js + FastAPI)

---

## 1. Обзор продукта

### 1.1 Проблема

Пользователь не понимает своё финансовое состояние и не знает, с чего начать управление деньгами. Информация разрознена: бюджет, цели, налоговые вычеты — каждый блок требует отдельного поиска и экспертизы.

### 1.2 Решение

Веб-приложение с AI-ассистентом, который отвечает на вопрос **«Что мне сейчас делать с деньгами?»** с учётом персонального финансового профиля, анализа бюджета, целей и доступных налоговых вычетов РФ.

### 1.3 Целевая аудитория

| Параметр | Значение |
|----------|----------|
| Сегмент | Начинающий инвестор / семья без накоплений |
| Возраст | 25–40 лет |
| Боль | Нет системы, деньги «утекают», не знает про налоговые вычеты |
| Ожидание | Простой план действий на понятном языке |
| Устройство | Браузер на ПК и мобильном |

### 1.4 Критерии успеха MVP

| Метрика | Цель |
|---------|------|
| Завершение онбординга | ≤ 7 минут |
| Персонализация AI | Нешаблонный ответ в первом сообщении с учётом профиля |
| Retention D7 | ≥ 20% |
| Налоговый вычет найден | ≥ 60% пользователей с указанным доходом |

### 1.5 Принятые продуктовые решения (уточнения)

| Решение | Выбор |
|---------|-------|
| LLM | **DeepSeek** (API) |
| Аутентификация | **NextAuth.js** на фронте как **прокси** к FastAPI JWT |
| Расчёт налогов | **Детерминированные формулы** на бэкенде + **AI** для объяснений |
| Точка входа | **Лендинг** с CTA «Начать бесплатно» |
| Монетизация | **Только Free** — без Premium и биллинга в MVP |
| Email | **Не используется** — без напоминаний D3/D7 |

## 2. Scope MVP

### 2.1 В scope

1. Лендинг и регистрация/вход
2. Онбординг и финансовый профиль + Financial Health Score
3. Чат-ассистент с памятью (SSE-стриминг)
4. Базовый анализ бюджета (ручной ввод + CSV)
5. Конструктор финансовых целей
6. Налоговый советник РФ (формулы + AI-объяснения, RAG)

### 2.2 Вне scope (v1+)

- Инвестиционный портфель и биржевые данные
- Интеграция с банками и брокерами через API
- Автозаполнение декларации 3-НДФЛ
- Голосовой режим
- Пенсионный калькулятор
- Рыночный мониторинг
- Мобильное приложение
- Premium-тариф и биллинг (ЮKassa)
- Email-рассылки и напоминания

## 3. Пользовательские сценарии

### 3.1 Happy Path

```
Лендинг (/) → «Начать бесплатно»
    → Регистрация (Google OAuth или email)
    → /onboarding (3 шага, ≤ 7 мин)
    → Health Score + план из 3 шагов
    → /chat — первый ответ уже персонализирован
    → /tax — расчёт вычетов (формулы) + объяснение (AI)
    → /goals — создание первой цели
    → возврат через /dashboard
```

| Маршрут | Описание | Доступ |
|---------|----------|--------|
| `/` | Лендинг, CTA «Начать бесплатно» | Публичный |
| `/auth/login` | Вход (email + пароль, Google OAuth) | Публичный |
| `/auth/register` | Регистрация | Публичный |
| `/onboarding` | Анкета 3 шага | Только новые пользователи |
| `/dashboard` | Health Score, быстрые действия | Авторизован |
| `/chat` | AI-ассистент (главная ценность) | Авторизован |
| `/budget` | Транзакции, графики, AI-анализ | Авторизован |
| `/goals` | Список целей | Авторизован |
| `/goals/[id]` | Детали цели, сценарии | Авторизован |
| `/tax` | Налоговый советник | Авторизован |
| `/profile` | Редактирование профиля, удаление аккаунта | Авторизован |

## 4. Функциональные требования

### 4.1 Лендинг

**FR-L01.** Главная страница `/` содержит ценностное предложение, ключевые возможности MVP и кнопку **«Начать бесплатно»**.

**FR-L02.** CTA ведёт на `/auth/register` для неавторизованных и на `/onboarding` или `/dashboard` для авторизованных (если онбординг завершён — на `/dashboard`).

**FR-L03.** Страница адаптивна (мобильный + десктоп).

### 4.2 Онбординг и профиль

**FR-O01.** Анкета из 3 шагов с прогресс-баром.

**Шаг 1 — Базовая информация:**
- Имя, возраст
- Семейное положение: один / в паре / с детьми
- Регион РФ (выпадающий список)

**Шаг 2 — Финансы:**
- Ежемесячный доход после налогов (₽)
- Обязательные расходы (₽): ипотека/аренда, кредиты
- Переменные расходы (₽): еда, транспорт, подписки
- Накопления (₽, опционально)
- Кредитная нагрузка: да/нет + сумма платежа

**Шаг 3 — Цели и опыт:**
- Основная цель: подушка / квартира / пенсия / образование / другое
- Срок цели (лет)
- Опыт с инвестициями: нет / читал / торгую
- Отношение к риску: консервативный / умеренный / агрессивный

**FR-O02.** После завершения рассчитывается **Financial Health Score (0–100)** и показывается персональный план из 3 шагов.

**FR-O03.** Профиль сохраняется в БД и передаётся в контекст каждого AI-запроса.

#### Формула Health Score

| Компонент | Условие | Баллы |
|-----------|---------|-------|
| Норма сбережений | (Доход − Расходы) / Доход > 20% | +30 |
| Долговая нагрузка | Платежи по кредитам / Доход < 30% | +25 |
| Подушка безопасности | Накопления / Расходы в мес. > 3 | +25 |
| Наличие цели | Есть цель + срок | +20 |

### 4.3 Чат-ассистент

**FR-C01.** Интерфейс в стиле Claude/ChatGPT: история сообщений, поле ввода, стриминг ответа.

**FR-C02.** История хранится в PostgreSQL (`chat_sessions`, `messages`).

**FR-C03.** При каждом запросе в system prompt передаётся профиль пользователя.

**FR-C04.** Ответы AI рендерятся как Markdown (жирный, курсив, списки, таблицы).

**FR-C05.** SSE-стриминг (`stream: true`) — текст появляется по мере генерации.

**FR-C06.** Быстрые вопросы при пустом чате; кнопка «Новый чат» создаёт новую сессию.

**FR-C07.** Модель: **DeepSeek** через API. Агент: Finance Assistant (без RAG).

**FR-C08.** Дисклеймер в ответах об инвестициях: информация носит образовательный характер.

**FR-C09.** Вопросы о налогах — перенаправление в раздел `/tax`.

### 4.4 Бюджет

**FR-B01.** Ручной ввод транзакций: сумма, категория, дата, комментарий.

**FR-B02.** Категории: еда, транспорт, ЖКХ, подписки, развлечения, здоровье, прочее.

**FR-B03.** Таблица транзакций с фильтром по месяцу; сводка по категориям (столбчатый график).

**FR-B04.** Импорт CSV: форматы Сбербанк Online и Тинькофф.

**FR-B05.** Парсинг CSV → AI-категоризация (DeepSeek) → preview → подтверждение импорта.

**FR-B06.** AI-анализ по кнопке «Проанализировать» или автоматически при ≥ 10 транзакций.

**FR-B07.** Результат анализа: топ-3 категории, рекомендации по сокращению, сумма высвобождения в год.

### 4.5 Финансовые цели

**FR-G01.** Создание цели: название, целевая сумма, срок, ожидаемая доходность (по умолчанию 10%).

**FR-G02.** Расчёт ежемесячного взноса по формуле FV (future value).

**FR-G03.** Прогресс-бар: текущие накопления vs цель.

**FR-G04.** Сценарии: +10% к взносу, +1 год к сроку.

**FR-G05.** Подсказка AI: откуда взять деньги на взнос (на основе анализа бюджета).

**FR-G06.** Лимит Free: **1 активная цель** на пользователя.

### 4.6 Налоговый советник

**FR-T01.** Анкета: тип занятости, имущество, ипотека, обучение, лечение, ИИС, страхование, фитнес, благотворительность.

**FR-T02.** Расчёт вычетов — **детерминированные формулы на бэкенде** (`POST /api/tax/calculate`):

| Вычет | База расчёта (MVP) |
|-------|-------------------|
| Имущественный (ст. 220) | до 260 000 ₽ |
| Ипотечные проценты | до 390 000 ₽ |
| Социальный (лечение, обучение, фитнес) | до 150 000 ₽ базы |
| ИИС тип А | до 52 000 ₽/год |
| Страхование жизни | до 120 000 ₽ базы |

**FR-T03.** AI (Tax Agent + RAG) **объясняет** результаты расчёта: статьи НК РФ, документы, куда подавать, дедлайны.

**FR-T04.** RAG по `knowledge_chunks` (pgvector): выдержки НК РФ, инструкции ФНС.

**FR-T05.** Не давать советы по уклонению от налогов; рекомендовать бухгалтера для сложных случаев.

### 4.7 Профиль и аккаунт

**FR-P01.** Редактирование данных профиля с пересчётом Health Score.

**FR-P02.** Удаление аккаунта и всех ПДн (`/profile` → «Удалить аккаунт»).

**FR-P03.** Согласие на обработку ПДн при регистрации; ссылка на политику конфиденциальности.

## 5. Нефункциональные требования

### 5.1 Производительность

- Время ответа API (без AI): < 500 ms (p95)
- Первый токен AI-стрима: < 3 s (p95)

### 5.2 Безопасность и 152-ФЗ

| Требование | Реализация |
|------------|------------|
| Шифрование | HTTPS (TLS 1.3), пароли bcrypt (cost 12) |
| Аутентификация | JWT access (15 мин) + refresh (30 дней, httpOnly cookie) |
| Авторизация | User читает только свои данные |
| SQL | SQLAlchemy ORM, без raw SQL |
| Хранение | Серверы в РФ (VPS / Yandex Cloud) |
| ПДн | Политика конфиденциальности, согласие, удаление аккаунта |
| Логирование AI | user_id + tokens, без содержимого сообщений |

### 5.3 Архитектура аутентификации

```
Browser → NextAuth.js (Next.js)
              ↓ credentials / Google OAuth
         FastAPI /api/auth/*
              ↓ JWT access + refresh
         NextAuth session (прокси токенов)
              ↓ Authorization: Bearer
         FastAPI protected endpoints
```

NextAuth **не хранит** бизнес-данные — только сессию и проксирует JWT к FastAPI.

### 5.4 AI-агенты (DeepSeek)

| Агент | Файл prompt | RAG | Стриминг |
|-------|-------------|-----|----------|
| Finance Assistant | `finance_assistant.txt` | Нет | SSE |
| Tax Agent | `tax_agent.txt` | Да (tax_*) | SSE |
| Budget Analysis | `budget_agent.txt` | Нет | SSE |

### 5.5 Тариф Free (единственный в MVP)

| Возможность | Лимит |
|-------------|-------|
| Сообщения в чате | 50 / месяц |
| Анализ бюджета | Ручной ввод + CSV |
| Финансовые цели | 1 активная |
| Health Score | Да |
| Налоговый советник | Полный доступ |

## 8. Зависимости и риски

| Риск | Митигация |
|------|-----------|
| Качество DeepSeek на русском финансовом контексте | Тестирование prompts, fallback-дисклеймеры |
| Неточность AI в налогах | Формулы на бэкенде — источник истины; AI только объясняет |
| CSV форматы банков меняются | Парсеры Сбер/Тинькофф + ручная коррекция в preview |
| 152-ФЗ | Хостинг в РФ, политика ПДн, удаление аккаунта |
| Лимит 50 сообщений/мес | Счётчик в Redis или БД, UI-предупреждение при приближении к лимиту |

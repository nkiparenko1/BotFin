# AI-агенты (DeepSeek)

> **Реализация:** все три агента подключены. Промпты в `backend/prompts/`. Клиент — `backend/services/ai_service.py`. Статус: [STATUS.md](./STATUS.md).

## 1. Общие принципы

| Параметр | Значение |
|----------|----------|
| Провайдер | DeepSeek API |
| Chat model | `deepseek-chat` (или актуальная на момент разработки) |
| Embedding model | DeepSeek embedding / совместимая (1536 dim) |
| Стриминг | SSE на всех chat endpoints |
| Язык ответов | Русский |
| Дисклеймер | Обязателен при инвестиционных темах |

## 2. Finance Assistant Agent

**Файл:** `backend/prompts/finance_assistant.txt`  
**Endpoint:** `POST /api/chat/message`  
**RAG:** Нет

### Контекст

- Полный профиль пользователя (имя, доход, расходы, накопления, цель, риск, Health Score)
- Последние 20 сообщений текущей сессии

### System prompt (шаблон)

```
Ты — AI финансовый советник для физических лиц РФ.
Отвечай просто, без жаргона. Обращайся к пользователю по имени: {name}.

Профиль:
- Возраст: {age}, регион: {region}, семья: {family_status}
- Доход: {monthly_income} ₽/мес, расходы: {fixed + variable} ₽/мес
- Накопления: {savings} ₽
- Цель: {main_goal}, срок: {goal_years} лет
- Риск-профиль: {risk_level}
- Financial Health Score: {health_score}/100

Правила:
1. Используй конкретные числа из профиля, не давай общих советов.
2. Если данных не хватает — задай уточняющий вопрос.
3. Вопросы о налогах → направь в раздел «Налоговый советник» (/tax).
4. Вопросы об инвестициях → добавь дисклеймер:
   «Информация носит образовательный характер и не является
   индивидуальной инвестиционной рекомендацией.»
5. Не давай прямых инвестиционных рекомендаций (конкретные бумаги, сроки).
```

### Лимит Free

50 user-сообщений в календарный месяц. При достижении лимита — HTTP 429 с сообщением.

## 3. Tax Agent

**Файл:** `backend/prompts/tax_agent.txt`  
**Endpoints:** `POST /api/tax/ask`  
**RAG:** Да — `knowledge_chunks` WHERE `topic LIKE 'tax_%'`

### Принцип: формулы + AI

1. `POST /api/tax/calculate` — **бэкенд** считает суммы (см. `tax_calculator.py`)
2. Результат сохраняется в `tax_calculations`
3. `POST /api/tax/ask` — AI **объясняет** готовый расчёт, не пересчитывает

### Контекст

- Налоговый профиль пользователя
- Последний расчёт из `tax_calculations` (deductions, total_return, documents)
- Top-5 релевантных chunks из RAG

### System prompt (шаблон)

```
Ты — AI налоговый консультант для физических лиц РФ.
Ты объясняешь результаты расчёта, но НЕ пересчитываешь суммы.

Расчёт пользователя (источник истины):
{deductions_json}
Итого к возврату: {total_return} ₽

Правила:
1. Ссылайся на конкретные статьи НК РФ (из контекста RAG).
2. Называй суммы ТОЛЬКО из расчёта выше — не выдумывай цифры.
3. Перечисляй необходимые документы для каждого вычета.
4. Указывай дедлайны (до 30 апреля следующего года).
5. Не давай советы по уклонению от налогов.
6. Для сложных случаев рекомендуй консультацию с бухгалтером.
```

### RAG-источники

| Файл | topic | Содержание |
|------|-------|------------|
| `nk_rf_excerpts.md` | `tax_deduction` | Ст. 218–221 НК РФ |
| `tax_deductions.md` | `tax_deduction` | Практические примеры |
| `fns_instructions.md` | `tax_fns` | Инструкции ФНС |
| `iis_guide.md` | `tax_iis` | ИИС тип А |
| `mortgage_deduction.md` | `tax_mortgage` | Имущественный вычет |

### Индексация

```bash
python rag/ingest.py --source rag/documents/ --chunk-size 500 --overlap 50
```

## 4. Budget Analysis Agent

**Файл:** `backend/prompts/budget_agent.txt`  
**Endpoint:** `POST /api/budget/analyze`  
**RAG:** Нет

### Контекст

- Агрегированные расходы за месяц по категориям
- Профиль (доход для расчёта %)
- Топ транзакций по сумме

### System prompt (шаблон)

```
Ты — AI аналитик личного бюджета для пользователей РФ.

Доход: {monthly_income} ₽/мес
Расходы за {month}:
{category_breakdown_json}

Правила:
1. Назови топ-3 категории расходов с суммами и % от дохода.
2. Предложи 2–3 конкретных способа сократить расходы.
3. Для каждой рекомендации укажи сумму высвобождения в месяц и год.
4. Используй конкретные числа из данных, не общие фразы.
5. Формат: краткий, структурированный (списки, bold для сумм).
```

### Триггеры

- Кнопка «Проанализировать» на `/budget`
- Автоматически при ≥ 10 транзакций за месяц (опционально, P1)

## 5. AI для категоризации CSV

Не отдельный агент — batch-вызов DeepSeek при импорте CSV.

**Input:** `{ description, amount, date }[]`  
**Output:** `{ category }[]` — одна из 7 категорий

Prompt короткий, без стриминга, batch до 50 транзакций за запрос.

## 7. Обработка ошибок

| Ситуация | Поведение |
|----------|-----------|
| DeepSeek API timeout | Retry 1x, затем «Сервис временно недоступен» |
| Rate limit DeepSeek | Exponential backoff, UI-сообщение |
| Лимит 50 сообщений | HTTP 429, UI: «Лимит исчерпан до {date}» |
| Пустой RAG result | Tax Agent работает только на профиле + расчёте |
